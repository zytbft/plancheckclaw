### 🔍 Bug 排查流程（25 分钟完成）

1. **数据验证** (5 分钟) - Python 检查 tasks.json
2. **Git 对比** (5 分钟) - git log --oneline path/to/file
3. **最小化修改** (10 分钟) - 删除"优化"，恢复简单
4. **验证** (5 分钟) - 编译 + 重启 + 测试

**禁止**: 未验证数据就直接修改代码

### ✨ SwiftUI 响应式黄金法则

```swift
// ✅ 正确：每次都重新计算
private var filteredTodayTasks: [TaskItem] {
    switch taskFilter {
    case .all: return todayTasks  // 从 @Published 获取
    ...
    }
}

// ❌ 错误：破坏响应式
@State private var cachedFilteredTasks: [TaskItem] = []
```

**原则**: `@Published` + 计算属性 = 自动刷新，不要手动缓存

### 📋 代码审查清单（修改前必问）

- [ ] 数据验证：我验证了数据文件吗？
- [ ] Git 对比：我对比了 Git 历史吗？
- [ ] 问题根源：我知道为什么会这样吗？
- [ ] 全局搜索：我 grep 了所有引用点吗？
- [ ] 一次性改完：我能一次性修改所有点吗？
- [ ] 真的需要吗：我真的需要这个"优化"吗？
- [ ] 删除测试：如果删除这段代码，会发生什么？
- [ ] 简单优于复杂：有更简单的方案吗？

### ⚠️ 三大陷阱

| 陷阱 | 特征 | 应对策略 |
|------|------|----------|
| **盲目修改** | "可能是 X 问题吧..." | STOP → CHECK → COMPARE → FIX |
| **过度优化** | "这里可以加缓存" | 问 → 测 → 比 → 忍 |
| **忽视历史** | 不看 Git 就动手 | git log → 找原始 → 问为什么 |

### 📊 效率目标

- **Bug 排查**: 25 分钟内完成
- **编译次数**: 最多 2 次
- **Git 对比**: 遇到问题第一个动作
- **数据检查**: UI 问题必须先做
