## 📋 实现概述

在"全部任务"视图的每个任务卡片下方标签行中，新增"今日"标签，用于快速识别已加入"我的一天"的任务。

---

## ✨ 功能详情

### 显示效果

**已加入今日任务**:
```
待开始 | 深度工作 | C 级 | ☀️ 今日 | 创建 11:43
```

**未加入今日任务**:
```
待开始 | 深度工作 | C 级 | 创建 11:43
```

### 技术实现

#### 1. 修改文件

**文件**: `plancheck/plancheck/TaskRowView.swift`  
**修改位置**: 第 38-52 行（任务标签 HStack）

#### 2. 代码实现

```swift
HStack(spacing: 8) {
    statusCapsule           // 状态标签（待开始/进行中/已完成）
    contextCapsule          // 情境标签（深度工作/碎片任务/会议）
    priorityCapsule         // 优先级标签（A 级/B 级/C 级）
    
    // 今日标签 ← 新增
    if task.isInMyDay {
        Text("☀️ 今日")
            .font(.caption2)
            .padding(.horizontal, 6)
            .padding(.vertical, 2)
            .background(Color.orange.opacity(0.15))
            .foregroundColor(.orange)
            .clipShape(Capsule())
    }

    if task.carryOverCount > 0 {
        Text("转入\(task.carryOverCount)次")
            .font(.caption2)
            .foregroundColor(.secondary)
    }

    Text("创建 \(timeFormatter.string(from: task.createdAt))")
        .font(.caption2)
        .foregroundColor(.secondary)
}
```

## 🎨 设计规范

### 标签样式

| 属性 | 值 | 说明 |
|------|-----|------|
| **文字内容** | `☀️ 今日` | 太阳 emoji + "今日"文字 |
| **字体** | `.caption2` | 与现有标签保持一致 |
| **内边距** | `.padding(.horizontal, 6)`<br>`.padding(.vertical, 2)` | 水平 6pt，垂直 2pt |
| **背景色** | `Color.orange.opacity(0.15)` | 橙色 15% 透明度 |
| **前景色** | `.orange` | 标准橙色 |
| **形状** | `.clipShape(Capsule())` | 胶囊形状 |

### 标签位置

标签行元素顺序（从左到右）：
1. 状态标签（statusCapsule）
2. 情境标签（contextCapsule）
3. 优先级标签（priorityCapsule）
4. **今日标签** ← 新增（仅当 `isInMyDay = true` 时显示）
5. 转入次数标签（可选，仅当 `carryOverCount > 0` 时显示）
6. 创建时间标签

### 颜色语义

- **橙色**: 与"小太阳"图标颜色一致，表示"我的一天"
- **透明度 0.15**: 与其他标签的视觉风格统一
- **胶囊形状**: 与其他标签保持一致

## 🔧 技术细节

### 1. 数据源

使用 `TaskItem.isInMyDay` 布尔属性：
- `true`: 任务已加入"我的一天"
- `false` 或 `nil`: 任务未加入"我的一天"

### 2. 条件渲染

使用 Swift 的 `if` 语句进行条件判断：
```swift
if task.isInMyDay {
    // 显示今日标签
}
```

### 3. 组件复用

复用了现有的标签样式模式：
- 与 `statusCapsule`、`contextCapsule`、`priorityCapsule` 相同的结构
- 统一的字体、间距、颜色方案

### 4. 无交互设计

该标签仅为**状态指示器**，不需要点击交互功能：
- 没有 `.onTapGesture` 修饰符
- 不会触发任何动作
- 纯视觉提示

## 📊 影响范围

### 影响的视图

由于 `TaskRowView` 是通用组件，此修改会影响以下视图：

1. **今日任务视图** (`TodayView.swift`)
   - 所有任务都会显示"今日"标签（因为本身就是今日任务）
   - ✅ 符合预期，强化"今日"概念

2. **全部任务视图** (`AllTasksView.swift`)
   - 只有已加入"我的一天"的任务显示标签
   - ✅ 主要目标视图，方便快速识别

3. **重要任务视图** (`ImportantView.swift`)
   - 如果重要任务同时在今日列表中，也会显示标签
   - ✅ 提供额外的上下文信息

### 性能影响

- **渲染开销**: 每个任务行增加一个条件判断和可能的标签渲染
- **内存占用**: 忽略不计（仅增加少量视图层级）
- **滚动性能**: 无明显影响（标签为简单文本视图）

## ⚠️ 注意事项

### 1. 今日任务视图的显示

在"今日任务"视图中，所有任务都会显示"今日"标签：
- 这是**符合预期**的行为
- 强化了"这是今天的任务"的概念
- 不会造成混淆

### 2. 标签长度

"☀️ 今日"包含 emoji 和文字：
- 在某些字体下可能占用较多空间
- 如果任务标签过多，可能会换行
- 建议后续优化：考虑屏幕宽度自适应

### 3. 颜色可访问性

橙色对于色弱用户可能不够明显：
- 当前阶段保持设计一致性
- 后续可在设置中添加"高对比度标签"选项
