## ✨ 新增功能

### 左侧导航菜单视觉分组

优化了侧边栏菜单的视觉层次结构，通过分隔线将不同类型的功能模块清晰分开。

#### 分组结构

```
基础功能模块:
├── 今日任务
├── 全部任务
├── 目标看板
├── 天人合一
├── 重要任务
├── 历史记录
└── 每周报告
───────────────  ← 分隔线 1

高级功能模块:
├── ⚡ 超级 Musk
└── 🎯 人生系统
───────────────  ← 分隔线 2

其他:
└── 回收站
```

## 🛠️ 技术实现

### 修改文件
- `plancheck/plancheck/ContentView.swift`

### 关键代码变更

**位置**: 左侧导航 List 结构

**变更内容**:
1. 移除了原有的 `Section` 包装和 "其他" 标题
2. 在"每周报告"菜单项后添加 `Divider()` 分隔线
3. 在"🎯 人生系统"菜单项后添加 `Divider()` 分隔线

```swift
// 之前:
Section {
    Label("⚡ 超级 Musk", systemImage: "brain.head.profile")
        .tag(Tab.muskAssistant)
    
    Label("🎯 人生系统", systemImage: "target")
        .tag(Tab.lifeSystems)
    
    Label("回收站", systemImage: "trash")
        .tag(Tab.trash)
} header: {
    Text("其他")
}

// 之后:
Label("每周报告", systemImage: "chart.bar.xaxis")
    .tag(Tab.weekly)

Divider()  // ← 分隔线 1

Label("⚡ 超级 Musk", systemImage: "brain.head.profile")
    .tag(Tab.muskAssistant)
        
Label("🎯 人生系统", systemImage: "target")
    .tag(Tab.lifeSystems)

Divider()  // ← 分隔线 2
        
Label("回收站", systemImage: "trash")
    .tag(Tab.trash)
```

## 📝 设计决策

### 为什么选择 Divider() 而不是 Section?

1. **简洁性**: 不需要额外的 header 标签
2. **灵活性**: 可以自由控制分组，不受 Section 约束
3. **视觉效果**: 分隔线更加明显和直观
4. **代码量**: 更少的代码嵌套层级

### 分组逻辑

- **第一组（基础功能）**: 高频使用的任务管理功能
- **第二组（高级功能）**: 深度思考和战略规划工具
- **第三组（其他）**: 辅助功能（回收站）

这种分组方式符合用户的心智模型和使用频率。
