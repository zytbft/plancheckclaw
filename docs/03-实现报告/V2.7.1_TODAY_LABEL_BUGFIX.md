## 🔍 问题描述

**现象**：
- 在"全部任务"视图中，即使点击小太阳图标，任务标签行也不显示"☀️ 今日"标签
- 页面 UI 与修改前完全一致
- 已重新编译并重启应用，问题依旧

**预期效果**：
- 已加入"我的今日"的任务应显示：`待开始 | 深度工作 | C 级 | ☀️ 今日 | 创建 11:43`
- 未加入的任务不显示"今日"标签

---

##  根本原因分析

### 1. 代码审查结果

#### TaskRowView.swift（第 44-53 行）

```swift
// 今日标签
if task.isInMyDay {
    Text("☀️ 今日")
        .font(.caption2)
        .padding(.horizontal, 6)
        .padding(.vertical, 2)
        .background(Color.orange.opacity(0.15))
        .foregroundColor(.orange)
        .clipShape(Capsule())
        .help("这个任务已加入今天")
}
```

**结论**：✅ 代码逻辑正确，条件判断无误。

---

#### TaskModel.swift（第 15 行）

```swift
var isInMyDay: Bool  // 是否加入"我的一天"
```

**结论**：✅ 属性定义正确，默认为 `false`。

---

#### AllTasksView.swift（第 291 行）- 问题所在

**修改前的代码**：
```swift
onToggleMyDay: { addToToday(task: task) }
```

**问题分析**：
1. `addToToday(task:)` 函数的作用是：
   - 创建一个**新任务**（`createdAt = 今天`）
   - 将**原任务标记为已放弃**
   - **不会设置 `isInMyDay = true`**

2. 新创建的任务 `isInMyDay` 仍然是 `false`

3. 因此 `TaskRowView` 中的条件判断 `if task.isInMyDay` 永远不会为 `true`

---

#### TaskStore+Tasks.swift（第 217-243 行）

**已存在的正确方法**：
```swift
/// 添加到"我的今日"
func addToMyDay(taskID: UUID) {
    guard let index = tasks.firstIndex(where: { $0.id == taskID }) else { return }
    guard !tasks[index].isInMyDay else { return }
    
    var updatedTask = tasks[index]
    updatedTask.isInMyDay = true
    updatedTask.updatedAt = Date()
    
    tasks[index] = updatedTask
    persist()
}

/// 从"我的今日"移除
func removeFromMyDay(taskID: UUID) {
    guard let index = tasks.firstIndex(where: { $0.id == taskID }) else { return }
    guard tasks[index].isInMyDay else { return }
    
    var updatedTask = tasks[index]
    updatedTask.isInMyDay = false
    updatedTask.updatedAt = Date()
    
    tasks[index] = updatedTask
    persist()
}
```

**结论**：✅ `TaskStore` 中已有正确的切换方法。

---

### 2. 功能概念混淆

项目中有两个不同的"今日"概念：

| 功能 | 方法 | 效果 | 使用场景 |
|------|------|------|---------|
| **添加到今日任务** | `addToToday(task:)` | 创建新任务（`createdAt=今天`），原任务标记为已放弃 | 将历史任务复制到今天执行 |
| **加入我的今日** | `addToMyDay(taskID:)` | 设置 `isInMyDay = true` | 标记"今日计划"任务 |

**问题根源**：
- `AllTasksView` 的小太阳图标错误地使用了 `addToToday`
- 应该使用 `addToMyDay` / `removeFromMyDay` 来切换状态

---

### 3. TodayView 的筛选逻辑

**TodayView.swift（第 52-53 行）**：
```swift
private var todayTasks: [TaskItem] {
    taskStore.sortedTodayTasks(referenceDate: Date())
}
```

**TaskStore+Tasks.swift（第 420-426 行）**：
```swift
func sortedTodayTasks(referenceDate: Date = Date()) -> [TaskItem] {
    let day = Calendar.current.startOfDay(for: referenceDate)
    let currentDayTasks = tasks.filter {
        Calendar.current.isDate($0.createdAt, inSameDayAs: day)
    }
    return taskPriorityService.sortTodayTasksByPriority(currentDayTasks)
}
```

**结论**：
- "今日任务视图"显示的是**创建日期为今天的任务**
- **与 `isInMyDay` 字段无关**

---

### 4. isInMyDay 的真实用途

**TaskStore+Tasks.swift（第 437-445 行）**：
```swift
/// 获取"我的今日"推荐任务
func myDaySuggestions(referenceDate: Date = Date(), limit: Int = 5) -> [TaskItem] {
    let day = Calendar.current.startOfDay(for: referenceDate)
    return tasks
        .filter {
            Calendar.current.isDate($0.createdAt, inSameDayAs: day)
                && $0.status.isPending
                && !$0.isInMyDay  // ← 筛选未加入"我的今日"的任务
        }
        .sorted { ... }
}
```

**结论**：
- `isInMyDay` 用于标记"用户手动选择的今日计划任务"
- 在"我的今日"推荐列表中，会排除已加入的任务

## ✅ 修复方案

### 修复内容

#### 1. 修改 AllTasksView.swift（第 291 行）

**修改前**：
```swift
onToggleMyDay: { addToToday(task: task) }
```

**修改后**：
```swift
onToggleMyDay: { 
    if task.isInMyDay {
        taskStore.removeFromMyDay(taskID: task.id)
    } else {
        taskStore.addToMyDay(taskID: task.id)
    }
}
```

**说明**：
- 点击小太阳图标时，切换 `isInMyDay` 状态
- 如果已加入，则移除
- 如果未加入，则添加

---

#### 2. 更新帮助文本

**修改前**：
```swift
sunIconHelpText: "添加到今日任务"
```

**修改后**：
```swift
sunIconHelpText: task.isInMyDay ? "从我的今日移除" : "加入我的今日"
```

**说明**：
- 根据当前状态动态显示提示文字
- 已加入时显示"从我的今日移除"
- 未加入时显示"加入我的今日"

## ⚠️ 注意事项

### 1. 功能定位

**"添加到今日任务"vs"加入我的今日"**：

- **添加到今日任务**（创建新任务）：
  - 适用于：将历史任务**复制到今天执行**
  - 操作位置：右键菜单、批量操作菜单
  - 效果：原任务标记为已放弃，新任务创建在今天

- **加入我的今日**（切换状态）：
  - 适用于：标记**今日计划任务**
  - 操作位置：小太阳图标
  - 效果：设置 `isInMyDay = true`，显示"今日"标签

---

### 2. 数据一致性

**修改后**：
- 原任务不会被标记为已放弃
- 不会创建新任务
- 只是切换 `isInMyDay` 状态

**影响**：
- ✅ 任务历史保持完整
- ✅ 不会增加重复任务
- ✅ 更符合"标记今日计划"的语义

---

### 3. 与其他功能的协调

**需要检查的功能**：
1. 批量操作菜单中的"添加到今日任务"
2. 右键菜单中的"添加到今日任务"
3. "我的今日"推荐列表

**建议**：
- 保持批量操作和右键菜单的 `addToToday` 功能不变
- 小太阳图标专门用于切换 `isInMyDay` 状态
