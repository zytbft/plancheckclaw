# V2.7.2 任务状态切换 Bug 修复

## 🐛 问题描述

在今日任务视图中，当用户点击任务行左侧的状态图标时：

- ✅ 点击**圆形图标**（待开始）→ 正确切换为**三角形图标**（进行中）
- ❌ 点击**三角形图标**（进行中）→ **无法切换回圆形图标**（待开始）

## 🔍 问题诊断

### 核心问题

在 `TaskRowView.swift` 第 326-330 行，状态切换逻辑错误：

```swift
case .inProgress:
    // 从"进行中"切换回"待处理"
    // 注意：这里使用 onReopen 回调，但实际逻辑应该是 pauseTask
    // 需要在调用方（TodayView/AllTasksView）中区分
    onReopen()  // ❌ 错误！调用了 reopenTask 而不是 pauseTask
```

### 问题分析

1. **`TaskRowView` 的逻辑错误**：
   - `.notStarted` → 调用 `onStart()` → ✅ 正确
   - `.inProgress` → 调用 `onReopen()` → ❌ **错误**

2. **`onReopen` 的语义错误**：
   - `reopenTask` 是将**已完成/已放弃**的任务重新打开为**进行中**
   - 而不是将**进行中**暂停为**待开始**

3. **正确的方法应该是**：
   - `.inProgress` → `.notStarted`：应该调用 `pauseTask()`

4. **视图刷新问题**：
   - `pauseTask` 方法缺少 `objectWillChange.send()` 调用
   - 参考之前修复"今日"标签的经验，需要强制触发 SwiftUI 视图刷新

    if isEditingMode {
        onToggleSelection()

#### 2. TaskStore+Tasks.swift

**修复 `pauseTask` 方法，添加视图刷新通知**

```swift
func pauseTask(taskID: UUID) {
    guard let index = tasks.firstIndex(where: { $0.id == taskID }) else { return }
    
    var updatedTask = tasks[index]
    updatedTask.status = .notStarted
    updatedTask.startedAt = nil
    updatedTask.endedAt = nil
    updatedTask.updatedAt = Date()
    
    tasks[index] = updatedTask
    
    // ✅ 强制触发 @Published 通知
    objectWillChange.send()
    
    persist()
}
```

## 📝 技术说明

### SwiftUI 视图刷新陷阱

SwiftUI 的 `@Published` 对于值类型（struct）的嵌套修改不够敏感：

```swift
// ❌ 错误：可能不会触发视图刷新
var updatedTask = tasks[index]
updatedTask.status = .notStarted
tasks[index] = updatedTask

// ✅ 正确：强制触发通知
var updatedTask = tasks[index]
updatedTask.status = .notStarted
tasks[index] = updatedTask
objectWillChange.send()  // 强制触发 @Published 通知
```

### 为什么需要 `objectWillChange.send()`？

1. `TaskItem` 是结构体（值类型）
2. 修改数组中的元素时，创建了副本
3. 虽然数组被替换了，但内存地址可能没有变化
4. SwiftUI 可能检测不到这种"原地替换"
5. 显式调用 `objectWillChange.send()` 确保视图刷新
