# PlanCheck v2.15 - 批量添加到今日任务数据同步修复

## 📅 更新时间

## 🐛 问题描述

### 用户报告的 Bug
在全部任务视图中，通过批量操作选中多个任务并执行"添加到今日任务"后：
- ✅ **现象 1**：这些任务在全部任务列表中正确显示了"☀️ 今日"标签
- ❌ **现象 2**：切换到今日任务视图时，这些新添加的任务并没有显示在列表中

### 预期行为
- ✅ 批量添加到今日任务后，任务立即显示"☀️ 今日"标签
- ✅ 切换到今日任务视图后，新添加的任务应该出现在列表中
- ✅ 无需手动刷新页面，数据自动同步

## 🔍 问题根因分析

### 数据流分析

#### 1. 全部任务视图的批量添加逻辑
**文件**: `AllTasksView.swift` 第 431-451 行

```swift
/// 批量添加到今日任务
private func addSelectedToToday() {
    var addedCount = 0
    for taskID in selectedTaskIDs {
        if let task = allTasks.first(where: { $0.id == taskID }) {
            // 如果任务还未加入"我的一天"，则加入
            if !task.isInMyDay {
                taskStore.addToMyDay(taskID: taskID)  // ← 调用 TaskStore 方法
                addedCount += 1
            }
        }
    }
    // ...
}
```

#### 2. TaskStore 的 addToMyDay 方法
**文件**: `TaskStore+Tasks.swift` 第 223-238 行

```swift
/// 添加到"我的今日"
func addToMyDay(taskID: UUID) {
    guard let index = tasks.firstIndex(where: { $0.id == taskID }) else { return }
    guard !tasks[index].isInMyDay else { return }

    // ✅ 创建新对象以触发 @Published 通知
    var updatedTask = tasks[index]
    updatedTask.isInMyDay = true
    updatedTask.updatedAt = Date()

    tasks[index] = updatedTask

    // ✅ 强制触发 @Published 通知
    objectWillChange.send()

    persist()
}
```

**分析**：这个方法正确更新了 `isInMyDay` 属性并触发了通知。

#### 3. 今日任务视图的数据源
**文件**: `TodayView.swift` 第 52-54 行

```swift
private var todayTasks: [TaskItem] {
    taskStore.sortedTodayTasks(referenceDate: Date())
}
```

#### 4. TaskStore 的 sortedTodayTasks 方法（❌ 问题所在）
**原代码**: `TaskStore+Tasks.swift` 第 441-447 行

```swift
/// 获取今天的任务（已排序）
func sortedTodayTasks(referenceDate: Date = Date()) -> [TaskItem] {
    let day = Calendar.current.startOfDay(for: referenceDate)
    let currentDayTasks = tasks.filter {
        Calendar.current.isDate($0.createdAt, inSameDayAs: day)  // ❌ 只筛选 createdAt 为今天的任务
    }
    return taskPriorityService.sortTodayTasksByPriority(currentDayTasks)
}
```

### 核心问题

**数据不一致**：
1. **全部任务视图**中的"☀️ 今日"标签显示的是 `isInMyDay == true` 的任务
2. **今日任务视图**筛选的是 `createdAt` 为今天的任务

**场景重现**：
- 用户在 3 月 28 日创建了一个任务（`createdAt = 3 月 28 日`）
- 用户在 4 月 1 日通过批量操作将该任务"添加到今日任务"
- `addToMyDay()` 方法将 `isInMyDay` 设置为 `true` ✅
- 但 `sortedTodayTasks()` 只筛选 `createdAt = 4 月 1 日` 的任务 ❌
- 结果：该任务不会出现在今日任务视图中

### 设计矛盾

**业务逻辑冲突**：
- **设计初衷 1**：今日任务视图应该显示用户标记为"今日要做的任务"（`isInMyDay == true`）
- **设计初衷 2**：今天创建的任务自动加入今日任务（`createdAt == today`）
- **当前实现**：只实现了设计初衷 2，忽略了设计初衷 1

## ✅ 修复方案

### 方案选择

对比两种修复方案：

| 方案 | 修改内容 | 优点 | 缺点 |
|------|---------|------|------|
| **方案 1** | 修改 `sortedTodayTasks` 筛选逻辑 | 兼容两种使用场景，不破坏时间戳记录 | 需要更新注释说明 |
| **方案 2** | 修改 `addToMyDay` 同时更新 `createdAt` | 实现简单 | 破坏了任务的原始创建时间，可能导致其他问题 |

**选择**：✅ **方案 1** - 更合理、更安全、更符合业务逻辑

### 修复实现

**文件**: `TaskStore+Tasks.swift` 第 441-449 行

**修复前**：
```swift
/// 获取今天的任务（已排序）
func sortedTodayTasks(referenceDate: Date = Date()) -> [TaskItem] {
    let day = Calendar.current.startOfDay(for: referenceDate)
    let currentDayTasks = tasks.filter {
        Calendar.current.isDate($0.createdAt, inSameDayAs: day)
    }
    return taskPriorityService.sortTodayTasksByPriority(currentDayTasks)
}
```

**修复后**：
```swift
/// 获取今天的任务（已排序）
/// 包含：1) isInMyDay == true 的任务；2) createdAt 为今天的任务
func sortedTodayTasks(referenceDate: Date = Date()) -> [TaskItem] {
    let day = Calendar.current.startOfDay(for: referenceDate)
    let currentDayTasks = tasks.filter { task in
        // 条件 1: isInMyDay == true（手动加入今日的任务）
        task.isInMyDay || 
        // 条件 2: createdAt 为今天（今天创建的任务自动加入）
        Calendar.current.isDate(task.createdAt, inSameDayAs: day)
    }
    return taskPriorityService.sortTodayTasksByPriority(currentDayTasks)
}
```

### 修复效果

**修复后的行为**：
1. ✅ 今日任务视图会显示所有 `isInMyDay == true` 的任务（无论何时创建）
2. ✅ 今日任务视图会自动包含今天创建的任务（即使没有手动标记）
3. ✅ 批量添加到今日任务后，切换视图立即可见
4. ✅ 数据自动同步，无需手动刷新

## 🎯 技术亮点

### 1. 兼容性设计
修复后的逻辑同时支持两种场景：
- **手动加入**：用户主动标记为今日任务（`isInMyDay == true`）
- **自动加入**：今天创建的任务自动成为今日任务（`createdAt == today`）

### 2. 数据完整性
- 不修改任务的 `createdAt` 时间戳
- 保留任务的原始创建时间信息
- 通过 `isInMyDay` 字段灵活管理今日任务

### 3. SwiftUI 响应式
- `TaskStore` 使用 `@Published var tasks`
- `addToMyDay` 方法中手动调用 `objectWillChange.send()`
- 确保视图自动刷新

### 4. 代码可读性
- 清晰的注释说明筛选条件
- 使用 `||` 逻辑运算符明确表达"或"关系
- 变量命名清晰（`task` 替代匿名闭包参数）
