## 产品简介

PlanCheck 是一款智能任务管理应用，帮助你：
- ✨ **优先级排序**：基于价值、成功率、杠杆率自动计算任务优先级
- 📅 **每日规划**：Top 3 必做任务推荐，避免拖延
- 🔄 **自动流转**：未完成的任务自动转入次日
- ⏱️ **时间估算**：智能建议任务预估时长
- 📊 **数据同步**：支持 iCloud 和文件夹同步

## 快速入门

### 1. 添加第一个任务

1. 打开应用，进入「今日任务」页面
2. 点击「批量粘贴」按钮
3. 输入任务内容（每行一个任务）
4. 按回车键确认，系统自动分析并创建任务

**示例输入：**
```
完成项目需求文档 | 60
回复客户邮件 | 15
团队周会 | 45
```

### 2. 开始执行任务

1. 点击任务左侧的圆形图标（⭕）开始任务
2. 任务状态变为「进行中」（🔵）
3. 完成后点击「完成」按钮
4. 记录实际用时，系统自动分析偏差

### 3. 查看优先级

系统会自动为任务标注优先级：
- **A 级**（红色星标⭐）：高价值、高概率任务
- **B 级**（橙色半星🌟）：中等优先级
- **C 级**（灰色圆圈⚪）：常规任务

## 核心功能详解

### 任务管理

#### 任务状态流转

```
待开始 → 进行中 → 已完成
              ↘
               已放弃
```

- **待开始**：未执行的任务
- **进行中**：正在执行的任务（只能有一个）
- **已完成**：完成的任务
- **已放弃**：取消的任务（可在回收站恢复）

#### 任务属性

每个任务包含以下信息：

| 属性     | 说明                             |
| -------- | -------------------------------- |
| 标题     | 任务描述（必填）                 |
| 预计时长 | 预估需要的分钟数（15-1440 分钟） |
| 情境     | 深度工作 / 碎片任务 / 会议       |
| 优先级   | A/B/C 三级，自动计算             |
| 重要标记 | 星标任务（独立于优先级）         |
| 置顶标记 | 固定在列表顶部                   |
| 我的一天 | 加入今日重点关注列表             |

#### 编辑任务

1. **快速编辑**：点击任务标题区域
2. **修改属性**：在弹出的编辑框中调整
3. **回车保存**：按回车键立即保存

**支持修改的属性：**
- 任务标题
- 预计时长
- 任务情境
- 备注说明
- 重要标记

### 优先级系统

#### 优先级计算维度

系统根据以下因素自动计算优先级分数（0-1）：

1. **预期收益** (`expectedCashInRMB`)
   - 任务成功后可兑现的金额
   - 数值越高，优先级越高

2. **到账周期** (`expectedDaysToCash`)
   - 从完成任务到获得收益的天数
   - 周期越短，优先级越高

3. **成功概率** (`successProbability`, 0-1)
   - 任务成功的可能性
   - 概率越高，优先级越高

4. **杠杆系数** (`leverageScore`, 1-5)
   - 任务的复制性和放大效应
   - 5 分表示一次投入多次回报

5. **战略匹配** (`strategicFitScore`, 1-5)
   - 与长期目标的一致性
   - 5 分表示完全匹配

#### 优先级分层规则

```swift
A 级：priorityScore >= 0.7  (红色，实心星)
B 级：0.3 <= score < 0.7    (橙色，半星)
C 级：score < 0.3           (灰色，圆圈)
```

#### 手动调整优先级

虽然系统会自动计算，但你可以通过以下方式影响排序：

1. **标记重要**：点击星标按钮（⭐）
2. **置顶任务**：点击图钉按钮（📌）
3. **加入我的一天**：点击太阳图标（☀️）

### 今日任务

#### 页面布局

「今日任务」页面分为三个区域：

**1. 今日概览区**
- 任务总数
- 计划用时（所有任务预计时长总和）
- 实际用时（已完成任务的实际时长总和）
- 批量粘贴按钮（快速添加任务）

**2. Top 3 必做区**
- 显示今天优先级最高的 3 个 A 级任务
- 仅显示待开始和进行中的任务
- 帮助你聚焦最重要的事情

**3. 任务列表区**
- 显示今天的所有任务
- 支持筛选和搜索
- 可滚动查看更多

#### 筛选器

点击筛选器下拉菜单，可选择：

- **全部**：显示所有今日任务
- **待处理**：未开始的任务
- **进行中**：正在执行的任务
- **已完成**：已完成的任务
- **已放弃**：已放弃的任务
- **重要**：标记了星标的任务

#### 全局搜索

在顶部搜索框输入关键词：
- 实时过滤任务列表
- 搜索范围：标题 + 备注
- 支持多词组合搜索（用空格分隔）

**示例：**
```
需求文档      # 搜索包含“需求文档”的任务
邮件 会议     # 同时包含“邮件”和“会议”的任务
```

### 全部任务

#### 与今日任务的区别

| 特性       | 今日任务       | 全部任务         |
| ---------- | -------------- | ---------------- |
| 显示范围   | 今天创建的任务 | 所有未放弃的任务 |
| 排序方式   | 优先级排序     | 创建时间倒序     |
| Top 3 展示 | ✅ 有           | ❌ 无             |
| 添加到今日 | ❌ 不支持       | ✅ 支持           |

#### 添加到今日任务

在「全部任务」中，可以将历史任务重新激活：

1. 找到需要重新执行的任务
2. 点击太阳图标（☀️）或右键选择「添加到今日任务」
3. 系统会：
   - 创建一个新任务（创建时间为今天）
   - 将原任务标记为已放弃
   - 保留原任务的备注和属性

**应用场景：**
- 之前放弃的任务想重新捡起
- 周期性任务需要再次执行
- 历史任务模板复用

### 批量操作

#### 进入编辑模式

1. 点击「批量操作」按钮
2. 界面切换到多选模式
3. 任务左侧出现选择圈

#### 选择任务

- **单选**：直接点击任务
- **连选**：按住 Shift 键，点击首尾任务（连续范围）
- **点选**：按住 Command 键，逐个点击（不连续）
- **右键选择**：右键点击任务，选择「选择此项」

#### 批量操作菜单

进入编辑模式后，点击「批量操作▼」菜单：

**1. 删除所选**
- 将选中任务移至回收站
- 可在 10 秒内撤销
- 回收站中可永久删除或恢复

**2. 放弃所选**
- 将选中任务标记为已放弃
- 可在 10 秒内撤销
- 任务从默认视图消失（可在「已放弃」筛选查看）

**3. 复制任务**
- 复制选中任务的标题和备注到剪贴板
- 多个任务用分隔符连接
- 方便分享给他人或导出

#### 退出编辑模式

点击「完成批量操作」按钮：
- 退出多选模式
- 清除所有选中状态
- 返回正常浏览模式

## 高级功能

### 任务转入机制

#### 什么是任务转入？

当任务在创建当天未完成时，系统会自动将其「转入」到第二天，避免任务遗漏。

#### 转入规则

**触发条件：**
1. 任务创建于昨天或更早
2. 状态为「待开始」或「进行中」
3. 转入次数未达到上限（默认最多转入 3 天）
4. 今天尚未被转入过

**转入逻辑：**
- 创建一个新任务（标题、属性相同）
- 新任务的创建时间为今天
- `carryOverCount`（转入次数）+ 1
- 原任务保留，标记最近转入时间

**示例：**
```
原始任务（3 月 20 日创建，carryOverCount: 0）
  ↓ 转入
新任务（3 月 21 日创建，carryOverCount: 1）
  ↓ 转入
新任务（3 月 22 日创建，carryOverCount: 2）
```

#### 午夜自动转入

**触发时间：** 每天午夜 00:00

**执行流程：**
1. 检查所有历史任务
2. 筛选出符合转入条件的任务
3. 批量创建新任务到今天
4. 更新原任务的转入记录

**手动转入：**
如果需要在非午夜时间转入，可以：
1. 在「全部任务」中找到历史任务
2. 点击「添加到今日任务」
3. 系统会创建新任务到今天

### 超时处理

#### 什么是超时？

当任务的 `actualMinutes`（实际用时）超过 `estimatedMinutes`（预计用时）时，视为超时。

#### 超时处理策略

在完成任务时，可以选择三种处理方式：

**1. 直接完成** (`.complete`)
- 不创建额外任务
- 任务标记为已完成
- 适用于超时可接受的情况

**2. 拆分续做** (`.split`)
- 创建拆分任务，留在今天继续做
- 任务标题：`原标题（拆分续做）`
- 预计时长 = 超时分钟数（最少 15 分钟，最多 480 分钟）
- 适用于任务确实需要更多时间完成

**3. 延后到明天** (`.postpone`)
- 创建延后任务，移到明天执行
- 任务标题：原标题
- 预计时长 = 超时分钟数
- `carryOverCount + 1`
- 适用于今天时间不足，明天继续

#### 示例场景

假设任务「编写测试用例」预计 60 分钟，实际用了 90 分钟：

**超时分钟：** 90 - 60 = 30 分钟

**选择拆分续做：**
```
原任务：编写测试用例（已完成，90 分钟）
新任务：编写测试用例（拆分续做）（待开始，30 分钟）
```

**选择延后到明天：**
```
原任务：编写测试用例（已完成，90 分钟）
新任务：编写测试用例（待开始，明天创建，30 分钟，carryOverCount: 1）
```

### 自动复盘

#### 晨间复盘

**时间：** 每天早上 8:00

**内容：**
1. 回顾昨天的任务完成情况
2. 统计完成率、超时率
3. 自动生成今日 Top 3 推荐

#### 晚间复盘

**时间：** 每天晚上 22:00

**内容：**
1. 检查今日未完成的任务
2. 提示是否需要转入明天
3. 生成周报数据

#### 周报生成

**时间：** 每周日晚上

**内容包括：**
- 本周完成任务数
- 总用时分布
- 各情境任务占比
- 优先级分布
- 超时情况分析

**导出格式：** CSV 文件，可在 Excel 中打开分析

### 外部执行器

#### 什么是外部执行器？

通过 Webhook 调用外部服务，实现任务自动化。

#### 配置方法

1. 打开「设置」>「自动化」
2. 启用「外部执行器」
3. 填写 Webhook URL
4. 选择触发时机（任务开始/完成）

#### 请求格式

```json
{
  "taskId": "UUID",
  "title": "任务标题",
  "status": "inProgress",
  "startedAt": "2025-03-24T10:00:00Z",
  "context": "deepWork"
}
```

#### 应用场景

- **任务开始时**：
  - 发送通知到 Slack/飞书
  - 启动番茄钟计时器
  - 更新项目管理工具状态

- **任务完成时**：
  - 记录时间日志
  - 触发下一步工作流
  - 发送完成报告

### 使用技巧

#### 1. 快速添加任务

**技巧：** 使用批量粘贴，一次输入多个任务

```
写周报 | 30
回复邮件 | 15
产品评审会 | 60
```

系统会自动：
- 解析分隔符后的分钟数
- 建议使用历史中位数作为预估时间
- 按优先级排序

#### 2. 利用优先级排序

**技巧：** 优先完成 A 级任务

- 早上先查看「Top 3 必做」
- 集中精力完成 A 级任务
- C 级任务可以批量放弃或委托他人

#### 3. 合理使用「我的一天」

**技巧：** 标记当天重点关注的任务

- 不同于「今日任务」（按日期分组）
- 「我的一天」是跨日期的标记
- 适合标记需要特别关注的任务
- 每天早上自动重置

#### 4. 使用置顶功能

**技巧：** 固定重要但不紧急的任务

- 置顶任务始终显示在列表顶部
- 不受优先级和创建时间影响
- 适合标记长期关注的事项

#### 5. 回收站撤销

**技巧：** 误删任务可以快速恢复

- 删除任务后显示 10 秒倒计时 Toast
- 点击「撤销」立即恢复
- 超过 10 秒可在「回收站」手动恢复

#### 6. 复制任务内容

**技巧：** 快速分享任务给他人

- 单个任务：点击复制按钮（📄）
- 批量复制：选中多个任务后右键复制
- 格式：标题 + 备注（用分隔符连接）

## 常见问题

### Q1: 如何修改任务的预计时长？

**答：** 
1. 点击任务标题进入编辑模式
2. 修改「预计时长」字段
3. 按回车键保存

或者在批量输入时直接指定：
```
任务标题 | 45
```

### Q2: 为什么我的任务被自动转入了？

**答：** 
系统在午夜检查未完成的任务，如果满足以下条件会自动转入：
- 创建于昨天或更早
- 状态为待开始或进行中
- 转入次数未达上限（默认 3 次）

如果不想让任务自动转入，可以：
- 在睡前标记为已放弃
- 或者手动完成它

### Q3: 如何查看历史任务数据？

**答：**
1. 进入「全部任务」页面
2. 使用筛选器选择「已完成」
3. 或使用搜索功能查找特定任务
4. 数据会导出为 CSV 文件（在周报中）

### Q4: 优先级分数如何计算？

**答：**
系统使用以下公式计算：

```swift
priorityScore = (
  归一化 (expectedCashInRMB) * 0.3 +
  归一化 (1 / expectedDaysToCash) * 0.2 +
  归一化 (successProbability) * 0.3 +
  归一化 (leverageScore / 5) * 0.1 +
  归一化 (strategicFitScore / 5) * 0.1
)
```

你可以在设置中调整权重（高级功能）。

### Q5: 如何备份数据？

**答：**
有两种备份方式：

**iCloud 同步：**
1. 打开「设置」>「iCloud 同步」
2. 启用同步开关
3. 数据自动备份到 iCloud Drive

**文件夹同步：**
1. 打开「设置」>「文件夹同步」
2. 选择同步目录（如 Dropbox 文件夹）
3. 数据实时同步到该目录

### Q6: 任务的情境有什么用？

**答：**
情境用于分类任务类型，帮助你在不同场景下选择合适的任务：

- **深度工作**：需要整块时间、高度专注（如写作、编程）
- **碎片任务**：可以利用零碎时间完成（如回邮件、打电话）
- **会议**：与他人协作的时间段（如开会、讨论）

系统会根据情境给出不同的时间建议：
- 深度工作：建议 60-120 分钟
- 碎片任务：建议 15-30 分钟
- 会议：建议 30-60 分钟

### Q7: 如何删除已放弃的任务？

**答：**
1. 进入「全部任务」页面
2. 筛选器选择「已放弃」
3. 点击「批量操作」进入编辑模式
4. 选中要删除的任务
5. 点击「删除所选」
6. 在回收站中确认删除

### Q8: 能否自定义优先级规则？

**答：**
可以！在「设置」>「LLM 设置」中：

1. 启用 AI 优先级推断
2. 配置你的偏好（更看重收益 / 概率 / 杠杆）
3. 系统会使用 AI 模型重新计算优先级

或者在代码中自定义规则（需要开发能力）。
