```javascript
mcp_github_push_files({
  owner: "zytbft",           // ✅ GitHub 用户名
  repo: "plancheckclaw",     // ✅ 仓库名

```

**参数验证清单（必须逐项勾选）：**

- [ ] `owner` 是字符串 `"zytbft"`（不是 undefined）
- [ ] `repo` 是字符串 `"plancheckclaw"`（不是 undefined）
- [ ] `branch` 是字符串 `"main"`（不是 undefined）
- [ ] `message` 是字符串（不是 undefined/null）
- [ ] `files` 是**数组** `[...]`（不是字符串/null/undefined）
- [ ] 每个文件的 `path` 和 `content` 都有值

## 🆘 遇到问题时的正确处理

### 问题：GitHub API 报错
**症状**: `internal error: Not Found`

**正确处理**:
1. ✅ 检查远程仓库是否存在
2. ✅ 运行 `git remote -v` 确认
3. ✅ **向用户求助**："我需要使用 MCP GitHub 工具推送代码，但遇到了 [具体错误]。请您提供解决方案或指导。"

**错误处理**:
1. ❌ 使用 `git push` 替代
2. ❌ 自作主张尝试其他方法
3. ❌ 不报告就直接行动

### 问题：参数格式不确定
**症状**: 不知道 `files` 应该是数组还是字符串

**正确处理**:
1. ✅ 读取此检查清单
2. ✅ 查看标准模板
3. ✅ **向用户确认**

**错误处理**:
1. ❌ 猜测着试一下
2. ❌ 用错误的参数调用
3. ❌ 失败后才查文档
